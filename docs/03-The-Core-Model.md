# 02 · The Core Model

## Overview

Atlas is built on a deliberately small model.

It does not begin with works, projects, worlds, people or exhibitions.

It begins with Things.

Everything else emerges through description and relationship.

The purpose of the model is not to classify knowledge.

It is to represent it as simply as possible.

---

## The Core

Atlas stores only three fundamental concepts.

```
Thing

Attribute

Connection
```

Everything else is built from these.

Paths, patterns, views, publishing and organisation all emerge from the relationships between them.

---

## Thing

A Thing is the fundamental building block of Atlas.

Atlas makes no assumptions about what a Thing represents.

A Thing might represent:

- a painting
- a sculpture
- a person
- a place
- a project
- an exhibition
- a conversation
- an idea
- a document
- something that has not yet been defined

A Thing exists independently of any description.

It may exist with nothing more than its unique identity.

---

## Attributes

Attributes describe a Thing.

They are entirely optional.

Attributes consist of a name and a value.

Examples:

```
Medium
Painting

Material
Jesmonite

Status
Complete
```

Atlas does not prescribe attribute names.

The first time a builder creates an attribute, Atlas remembers it.

The next time an attribute is added, previously used names are suggested.

Attribute values behave in exactly the same way.

Vocabulary emerges through use rather than being predefined.

---

## Connections

Connections relate Things.

A Connection always exists between two Things.

Connections may exist without a label.

```
○────────○
```

Labels are optional descriptions added by the builder.

```
Merge 001

Exhibited at

SOMA
```

Connections may also have an optional reverse label.

```
Merge 001

Exhibited at

SOMA

↓

SOMA

Exhibited by

Merge 001
```

This allows relationships to read naturally from either direction while remaining a single Connection.

Connection labels emerge through use in the same way as attributes.

Atlas remembers previously used labels and suggests them when creating new Connections.

---

## Creating Connections

Connections may be created in different ways.

A builder might:

- drag one Thing onto another
- connect Things from the inspector
- select another Thing while describing a relationship

Regardless of how they are created, the underlying model remains identical.

---

## The Graph

Together, Things and Connections form a graph.

The graph is the source of truth.

Atlas does not attempt to impose hierarchy onto the graph.

Hierarchy, structure and organisation emerge from the relationships between Things.

---

## Views

Views do not change the graph.

They change how the graph is seen.

A View might:

- colour Things by Medium
- shape Things by Type
- size Things by number of Connections
- hide or show particular Attributes

The underlying graph remains unchanged.

---

## Emergence

As a graph grows, Atlas begins to reveal information that was never explicitly created.

Examples include:

- paths
- clusters
- patterns
- isolated Things
- heavily connected Things
- opportunities for organisation

Atlas does not invent these.

It reveals structures that already exist.

---

## Principles

The Core Model is governed by a small number of principles.

- Everything begins as a Thing.
- Description is optional.
- Relationships are first-class.
- Vocabulary emerges through use.
- The graph is the truth.
- Views are interpretations of the truth.
- Atlas stores knowledge once.

Everything else is built upon these ideas.
